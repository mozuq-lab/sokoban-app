# プロダクト要件メモ

> Cloud Code / AI エージェント向けの短い要件整理。
> この文書は、repo が何を作るものかを素早く理解するための入口として使う。

## これは何か

このリポジトリは、Flutter で作る個人用のモバイル倉庫番（Sokoban）アプリである。

- 対象: iOS / Android
- ジャンル: 箱を押してゴールに運ぶパズルゲーム
- 方針: シンプルで遊びやすく、最初は小さく正しく作る

## プロダクトの狙い

- 倉庫番として正しいルールで遊べること
- 最低限の気持ちよさがあること
- AI が継続的に開発しやすい、理解しやすい構造にすること

## v1 の完成ライン

v1 では、少なくとも次を含める。

- 複数ステージを遊べる
- 画面内ボタン（上下左右）で操作できる
- Undo がある
- Restart がある
- クリア判定がある
- スマホ実機がなくても、Flutter Web で基本動作を確認できる

## v1 の体験イメージ

- 見た目はミニマル寄り
- ただし無機質すぎず、少し気持ちよく触れる
- 操作や盤面表示は分かりやすさを優先する

## レベルデータ方針

- レベルはテキスト形式で `assets` に持つ
- 倉庫番と相性のよい、読み書きしやすい表現を優先する
- 将来フォーマットを拡張できても、v1 では複雑にしすぎない

## v1 で優先すること

- コアロジックの正しさ
- 複数ステージを無理なく扱える構造
- Undo / Restart を無理なく追加できる状態管理
- テストしやすいドメインロジック
- 小さく着実な実装単位

## v1 で入れないもの

少なくとも初期バージョンでは、次は対象外とする。

- クラウド同期
- レベルエディタ
- 派手なアニメーション
- オンライン要素

## 実装上の含意

AI エージェントは、以下を前提に判断してよい。

- 倉庫番の基本ルールを中心に設計する
- UI より先に、盤面・移動・箱押し・クリア判定を安定させる
- 複数ステージと Undo / Restart を見据えて、状態表現を雑にしない
- ただし、過剰設計は避ける
